# Vistamz P0 内测验收清单

## 验收前提

- `/api/health` 的 `internalBetaReadiness.ready` 为 `true`。
- 设计、运营、管理员三个测试账号可登录。
- 每个项目都明确分配设计负责人和运营审核人。
- 验收项目使用真实产品资料，不复用其他项目的图片或卖点。

## 三个品类

选择结构和使用方式差异明显的三个品类：

1. 家具或儿童用品：验证结构、比例、折叠或使用状态。
2. 厨具或家居用品：验证材质、容量、热性能或清洁卖点。
3. 工具或电子设备：验证配件、接口、操作方式和真实场景。

每个品类分别完成一套主图和一套 A+。

## 单项目完整流程

### 1. 创建与资料

- 选择有 SKU 或无 SKU 模式。
- 上传主参考图和至少一种补充角度或细节图。
- 不合格参考图会在规划前被拦截并说明补图要求。
- 卖点可以编辑、合并、删除，不会出现其他项目内容。
- 选择品牌时保存品牌版本快照；不选品牌时使用基线模式。

### 2. 图片方案

- 主图默认生成 7 个图槽，第 1 槽为白底主图。
- A+ 根据内容生成 4–7 个模块，不套用白底主图规则。
- 每个图槽都能说明主要卖点、画面证据和允许文案。
- 添加、删除、前后排序可选图槽后，编号连续且不会串图。
- 变更方案结构后，旧候选和旧审核状态不会错误保留。

### 3. 生图与恢复

- 当前图槽固定生成 3 张；整套固定每槽生成 1 张。
- 任务显示排队、生成中、AI 预审中、成功或失败。
- 重复点击不会创建重复计费任务。
- 刷新页面后仍能恢复任务和候选图。
- 多角度、状态、细节和尺寸图槽会使用对应参考图。
- 单次失败可自动重试，最终失败有可理解原因。

### 4. 选择与审核

- 设计师可以比较候选并为图槽选择一个版本。
- 必需图槽未全部选择时，不能提交运营。
- 运营只能看到分配给自己的项目，不能进入设计生图台。
- 运营可通过或退回，并绑定图槽、版本、问题类型和说明。
- 退回后只重做对应图槽，历史版本仍可追溯。
- 设计师不能审核自己的图片。

### 5. 导出与跨设备

- 所有必需图槽未通过运营审核时，不能导出。
- 导出 ZIP 只包含每槽最终通过版本。
- 文件名包含图槽序号和简短名称。
- API 重启后图片仍可访问，已审核图片可重新打包 ZIP。
- 换一台电脑登录后仍可恢复项目、候选、审核和导出。

## 每套记录

| 指标 | 记录内容 |
| --- | --- |
| 输出类型 | 主图 / A+ |
| 图槽数量 | 实际数量 |
| 生成成功率 | 成功张数 / 请求张数 |
| 总用时 | 从开始生成到完成候选 |
| AI 预审命中 | AI 提示中被人工确认的问题数 |
| 人工退回原因 | 产品一致性 / 物理逻辑 / 文案 / 品牌 / 审美 / 其他 |
| 最终可用率 | 通过图槽数 / 总图槽数 |
| 估算成本 | 管理员工作台当日记录 |
| 阻塞问题 | 是否影响完整闭环 |

## P0 通过标准

- 三个品类的主图和 A+ 均完成创建、生成、审核、导出。
- 跨重启、跨设备恢复通过。
- 权限越权测试全部被服务端拒绝。
- 无项目串图、图槽串图、品牌快照漂移或导出错版本。
- 失败任务可定位到项目、用户、图槽、任务 ID 和错误原因。
