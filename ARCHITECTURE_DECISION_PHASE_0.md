# Vistamz 阶段 0 架构决策

> 状态：作为内测版正式改造的默认技术方案。目标是支持小规模多人协作，同时保留后续扩展空间。

## 1. 决策摘要

Vistamz 继续使用一套公司管理的 Gemini 图像 API Key，但不再将项目、邀请码、日志和图片结果保存在浏览器或 Render 临时磁盘。

推荐架构：

```text
Vercel 前端
   |
   v
Node API 服务
   |---- Postgres：账号、项目、卖点、方案、任务、审核、日志
   |---- 对象存储：产品参考图、候选图、最终图、ZIP
   |---- Worker：生图、AI 预审、压缩、导出等异步任务
   |
   v
Gemini API（仅 Node 服务持有公司 Key）
```

## 2. 服务职责

### 前端

- 登录后只显示当前身份可见的工作台。
- 上传文件时先做基础校验，再通过带权限的上传地址保存到对象存储。
- 只向 Node API 请求业务数据和任务状态；不保存真实项目作为唯一数据源。
- 不持有 Gemini Key、管理员码明文或数据库连接信息。

### Node API

- 校验登录身份、项目归属、角色和操作权限。
- 保存项目数据、生成策略、品牌版本、审核记录与日志。
- 编译生图提示词合同并调用 Gemini API。
- 创建、查询、取消和重试任务；返回可展示的进度和失败原因。
- 签发图片和 ZIP 的受控访问地址。

### Postgres

作为全部业务事实来源，保存：

- 用户、角色、邀请码、账号激活记录。
- 项目、负责人、审核人、参考图元数据、产品锁、卖点。
- 品牌与版本快照、图片方案、图槽、生成任务、候选版本。
- AI 预审、人工审核、退回意见、导出记录、审计日志。

### 对象存储

保存二进制文件，不保存业务逻辑：

- 原始参考图和处理后的预览图。
- 候选图、局部修改版本和审核通过最终图。
- 仅在需要时生成的 ZIP 导出包。

对象路径按 `环境 / 项目 / 图槽 / 版本` 组织，禁止用公开可猜测 URL 作为长期权限方案。

### Worker

- 消费数据库任务，避免在网页请求里直接等待模型返回。
- 任务类型：`generate_image`、`pre_review`、`create_export`、`compress_preview`。
- 每个任务有唯一 ID、输入快照、重试次数、状态、错误类型、成本估算和输出地址。
- 首期可以与 API 部署在同一平台的独立 worker 服务中，后续再拆分。

## 3. 为什么不能继续用当前存储

- 浏览器 `localStorage` 不能在多名同事、不同电脑和不同身份之间共享，也无法可靠审计。
- Render 免费实例的文件系统会重置，不能作为候选图、邀请码激活状态或导出包的正式存储。
- 文件路径和浏览器内存不能承受生成任务异步、重试、版本回退和多人审核。

## 4. 认证与邀请码

- 使用正式账号登录，而非仅凭浏览器保留的邀请码状态。
- 一次性邀请码在首次激活时绑定账号 ID，并立即失效。
- 管理员账号不使用公开共享邀请码；管理员登录应有独立、可撤销的账户权限。
- 管理员码只可作为当前内测的临时引导机制，进入阶段 1 后迁移为管理员账户和服务端权限。

## 5. 环境划分

| 环境 | 用途 | 数据与密钥 |
| --- | --- | --- |
| 本地开发 | 功能开发和单人调试 | 本地数据库或独立开发库、开发 Key |
| 预览 | 每次发布前验收 | 独立预览数据库/存储前缀、受限 Key |
| 内测 | 真实同事使用 | 正式内测数据库、对象存储前缀、公司 Key |

- 环境间不得共用数据库 schema、对象存储前缀或邀请码记录。
- 前端只配置 API 地址；所有敏感变量只放在 API/worker 环境。

## 6. 安全边界

- Gemini/OpenAI Key 只存在服务端环境变量中。
- 所有对象存储读写通过后端鉴权和短期签名 URL 完成。
- 后端根据当前用户、项目成员关系和角色判断权限，不相信前端传来的角色字段。
- 审计日志记录身份、项目、动作、结果、任务 ID 和时间，不记录 API Key、完整图片 base64 或邀请码明文。

## 7. 第一期数据迁移策略

- 不自动迁移所有本地草稿；先提供“导入当前项目”的一次性工具，避免导入过期测试数据。
- 导入时迁移项目基础信息、卖点、品牌选择、方案、候选图引用和审核记录。
- 原有本地数据在迁移后保留只读备份，直到内测稳定。
- 生成图片应重新上传到对象存储并获得新的资产 ID，不继续依赖旧的 Render 文件 URL。

## 8. 阶段 0 不做的事

- 不重写现有 UI。
- 不立即接 ERP。
- 不更换图像生成模型。
- 不把“管理员码”误当成正式权限系统。

## 9. 阶段 0 完成条件

- 已创建源代码快照备份。
- 已确认业务数据、图片资产和异步任务的最终归属。
- 已确认公司统一 API Key 仍只在后端使用。
- 已确定阶段 1 按账号、权限、项目分配和日志开始实施。
